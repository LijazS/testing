Backing up an etcd cluster

Step 1: Find etcd certs & endpoints

sudo cat /etc/kubernetes/manifests/etcd.yaml | grep -E "cert|key|listen-client-urls"

Step 2: BACKUP (take snapshot)

if you dont have etcdctl installed:

sudo apt update
sudo apt install etcd-client -y

Replace the below if your credentials are different:

sudo ETCDCTL_API=3 etcdctl snapshot save /tmp/backup.db \
  --endpoints=https://127.0.0.1:2379 \ 			
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key


OR 

1. exec into etcd container that is running in your cluster.

sudo crictl ps | grep etcd

sudo crictl exec -it <container-id> sh

2. Then do the same command to take the snapshot mentioned above
